Apache เซิร์ฟเวอร์ HTTP อันดับหนึ่งบนอินเทอร์เน็ต

ที่มา : https://httpd.apache.org/

โครงการ Apache HTTP Server มุ่งพัฒนาและดูแลเซิร์ฟเวอร์ HTTP โอเพนซอร์ส สำหรับระบบปฏิบัติการสมัยใหม่ ทั้ง UNIX และ Windows เป้าหมายคือสร้างเซิร์ฟเวอร์ที่ปลอดภัย มีประสิทธิภาพ ยืดหยุ่น และส่งมอบบริการ HTTP ตามมาตรฐาน HTTP ล่าสุด Apache HTTP Server ("httpd") เปิดตัวครั้งแรกในปี 1995 และกลายเป็นเซิร์ฟเวอร์เว็บที่นิยมที่สุดบนอินเทอร์เน็ตตั้งแต่เดือนเมษายน 1996 ปี 2020 ฉลองครบรอบ 25 ปีในฐานะโครงการ โครงการ Apache HTTP Server อยู่ภายใต้การกำกับดูแลของมูลนิธิ Apache Software Foundation

โครงการ Apache HTTP Server คืออะไร?

โครงการ Apache HTTP Server เป็นชุมชนนักพัฒนาซอฟต์แวร์ร่วมมือกัน สร้างเซิร์ฟเวอร์เว็บ HTTP โอเพนซอร์ส ที่แข็งแกร่ง ฟีเจอร์ครบครัน เหมาะสำหรับใช้งานระดับองค์กร โครงการนี้อยู่ภายใต้การบริหารจัดการร่วมกันโดยกลุ่มอาสาสมัครทั่วโลก พวกเขาใช้เครือข่ายอินเทอร์เน็ตและเว็บไซต์ในการสื่อสาร วางแผน และพัฒนาเซิร์ฟเวอร์พร้อมเอกสารประกอบ โครงการนี้เป็นส่วนหนึ่งของมูลนิธิ Apache Software Foundation นอกจากนั้น ยังมีผู้ใช้งานอีกหลายร้อยคนที่สมทบไอเดีย โค้ด และเอกสารให้กับโครงการ

พัฒนาการของ Apache HTTP Server

ย้อนกลับไปในเดือนกุมภาพันธ์ปี 1995 ซอฟต์แวร์เซิร์ฟเวอร์ที่โด่งดังที่สุดบนเว็บไซต์คือ HTTP daemon โอเพนซอร์ส พัฒนาโดย Rob McCool จากมหาวิทยาลัย Illinois อย่างไรก็ตาม การพัฒนาหยุดชะงักหลัง Rob ออกจาก NCSA ในกลางปี 1994 ส่งผลให้เว็บมาสเตอร์หลายคนพัฒนาส่วนเสริมและแก้ไขบั๊กเองโดยขาดการกระจายข้อมูลร่วมกัน

กลุ่มเว็บมาสเตอร์ขนาดเล็ก ติดต่อกันผ่านอีเมลส่วนตัว มารวมตัวกันเพื่อประสานการเปลี่ยนแปลงเหล่านี้ (ในรูปแบบ "แพตช์") Brian Behlendorf และ Cliff Skolnick จัดทำเมล์ลิสต์ พื้นที่เก็บข้อมูลร่วม และล็อกอินสำหรับนักพัฒนาหลัก บนเครื่องในอ่าวแคลิฟอร์เนีย โดย HotWired สนับสนุนแบนด์วิธ

ภายในสิ้นเดือนกุมภาพันธ์ ผู้พัฒนาหลัก 8 คน กลายเป็นรากฐานของกลุ่ม Apache กลุ่มเดิม:

  • Brian Behlendorf
  • Roy T.Fielding
  • Rob Hartill
  • David Robinson
  • Cliff Skolnick
  • Randy Terbush
  • Robert S.Thau
  • Andrew Wilson

พร้อมด้วยผู้สนับสนุนเพิ่มเติม:

  • Eric Hagberg
  • Frank Peters
  • Nicolas Pioch

โดยใช้ NCSA httpd 1.3 เป็นพื้นฐาน เพิ่มการแก้ไขบั๊กที่เผยแพร่และการปรับปรุงที่มีคุณค่าทั้งหมดที่พบ ทดสอบผลลัพธ์บนเซิร์ฟเวอร์และเปิดตัว Apache เวอร์ชันแรกอย่างเป็นทางการ (0.6.2) ในเดือนเมษายน 1995 บังเอิญ NCSA เริ่มพัฒนาต่อในช่วงเวลาเดียวกัน Brandon Long และ Beth Frank จากทีมพัฒนาเซิร์ฟเวอร์ NCSA เข้าร่วมรายชื่อในเดือนมีนาคม เพื่อให้ทั้งสองโครงการสามารถแลกเปลี่ยนแนวคิดและการแก้ไข

เซิร์ฟเวอร์ Apache ในช่วงแรกประสบความสำเร็จอย่างมาก แต่ทุกคนรู้ว่าCodebaseต้องการการปรับปรุงและออกแบบใหม่โดยสิ้นเชิง ระหว่างเดือนพฤษภาคม-มิถุนายน 1995 ในขณะที่ Rob Hartill และกลุ่มอื่นๆ มุ่งเน้นไปที่การพัฒนาฟีเจอร์ใหม่สำหรับ 0.7.x (เช่น กระบวนการลูกก่อนการฟอร์ค) และสนับสนุนชุมชนผู้ใช้ Apache ที่เติบโตอย่างรวดเร็ว Robert Thau ออกแบบสถาปัตยกรรมเซิร์ฟเวอร์ใหม่ (ชื่อรหัส Shambhala) ซึ่งรวมถึงโครงสร้างโมดูลาร์และ API เพื่อความยืดหยุ่นที่ดีกว่า การจัดสรรหน่วยความจำแบบกลุ่ม และรูปแบบกระบวนการก่อนการฟอร์คแบบปรับเปลี่ยนได้ กลุ่มปรับเปลี่ยนไปใช้ฐานเซิร์ฟเวอร์ใหม่นี้ในเดือนกรกฎาคม และเพิ่มฟีเจอร์จาก 0.7.x ส่งผลให้เป็น Apache 0.8.8 (และรุ่นน้อง) ในเดือนสิงหาคม

หลังจากการทดสอบเบต้าอย่างเข้มข้น การพอร์ตไปยังแพลตฟอร์มลึกลับ คู่มือชุดใหม่ (โดย David Robinson) และการเพิ่มฟีเจอร์มากมายในรูปแบบโมดูลมาตรฐานของ Apache 1.0 เปิดตัวในวันที่ 1 ธันวาคม 1995

ไม่ถึงหนึ่งปีหลังจากก่อตั้ง กลุ่ม Apache เซิร์ฟเวอร์แซงหน้า NCSA's httpd ขึ้นเป็นเซิร์ฟเวอร์อันดับ 1 บนอินเทอร์เน็ต และจากการสำรวจโดย Netcraft ยังคงรักษานำโด่งจนถึงปัจจุบัน

ในปี 1999 สมาชิกของกลุ่ม Apache จัดตั้งมูลนิธิ Apache Software Foundation เพื่อสนับสนุนด้านองค์กร กฎหมาย และการเงินสำหรับ Apache HTTP Server มูลนิธิทำให้ซอฟต์แวร์มั่นคงสำหรับการพัฒนาในอนาคต และเพิ่มโครงการซอฟต์แวร์โอเพนซอร์สจำนวนมาก ภายใต้มูลนิธิ
